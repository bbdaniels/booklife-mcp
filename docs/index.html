BookLife MCP

Your reading life, unified. One MCP server that connects Hardcover (reading tracker), Libby (library access), and Open Library (metadata) into a seamless AI-powered reading assistant with 27 tools.

Library Access (Libby)

Search your library catalog, check availability, place holds on ebooks and audiobooks, and track loans and due dates.

Reading Tracker (Hardcover)

Manage your reading list, update progress, rate books, and maintain your reading history through natural language.

Unified TBR

A single to-be-read list aggregating Hardcover want-to-read, Libby holds and tags, and manually added physical books.

Comprehensive Sync

One-command sync: import Libby loans, mark returned books as read in Hardcover, enrich metadata, cache tags. Incremental with dry-run preview.

Recommendations

Content-based recommendations using enriched themes, topics, and mood data from Open Library and Google Books.

Reading Analytics

Format preferences, top genres, favorite authors, reading cadence, streaks, and completion rates -- all analyzed automatically.

Quick Start

# Set your Hardcover API key
export HARDCOVER_API_KEY="your-key-from-hardcover.app/settings/api"

# Add to Claude Code
claude mcp add booklife -e HARDCOVER_API_KEY=your-key -- booklife serve
Install from source: Requires Go 1.23+. Clone the repo, then cd booklife-mcp/booklife-mcp && make install to build and install to ~/.local/bin/booklife.

Environment Variables

Variable Required Description
HARDCOVER_API_KEY Yes API key from hardcover.app/settings/api
BOOKLIFE_DATA_DIR No Data directory (default: ~/.local/share/booklife)
BOOKLIFE_LOG_LEVEL No debug / info / warn / error

CLI Commands

booklife serve --config booklife.kdl    # Start MCP server
booklife libby-connect <code>           # Connect Libby account
booklife sync [--dry-run] [--limit N]   # Sync returned books to Hardcover
booklife import-timeline <file>         # Import Libby timeline JSON
booklife version                        # Show version
